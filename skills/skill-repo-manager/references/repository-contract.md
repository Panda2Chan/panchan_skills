`dist/` is generated and ignored by Git. Do not edit it manually.

## Index Schema

`dist/index.json` is generated by `scripts/build_site.py`.

Core fields:

- `schemaVersion`: index schema version.
- `generatedAt`: UTC build timestamp.
- `totalSkills`: number of packaged skills.
- `skills[].name`: skill folder name and `SKILL.md` frontmatter name.
- `skills[].description`: trigger-oriented description from `SKILL.md`.
- `skills[].version`: optional version from `skill.json`.
- `skills[].tags`: optional tags from `skill.json`.
- `skills[].package`: relative zip URL.
- `skills[].sha256`: zip checksum.
- `skills[].codexHomeTarget`: expected install target.

## Package Rules

- Include the skill folder as the zip top-level directory.
- Exclude `.git`, `__pycache__`, `__MACOSX`, `.DS_Store`, `Thumbs.db`, and
  compiled Python files.
- Keep deterministic ordering, timestamps, and file permissions when possible.

## Install Command Pattern

Use relative download URLs in the Web UI, then produce shell commands like:

```bash
target="${CODEX_HOME:-$HOME/.codex}/skills" && mkdir -p "$target" && curl -L "<zip-url>" -o "/tmp/skill-name.zip" && unzip -oq "/tmp/skill-name.zip" -d "$target"
```
